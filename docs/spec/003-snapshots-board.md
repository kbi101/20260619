# 003 Snapshots Board — Technical Specification

## Overview
Traders require an independent dashboard window to display image snapshots generated by other applications during the day. This document outlines the functional requirements and architectural specifications for the Snapshot Board feature.

---

## Requirements

### 1. Snapshot Folder Resolution
- Snapshots are stored locally on the host machine at `/Users/kepingbi/Data/QuantEdge/YYYYMMDD`.
- The application must dynamically resolve `YYYYMMDD` using the client's current local date in timezone-local time.
- If today's directory does not exist, the UI must show a clean, descriptive empty state and watch for directory creation in the background.

### 2. Category & Timestamp Parsing
- Snapshots are files named with the pattern: `[category]_[timestamp].[ext]`.
  - Example: `volatility_201207.png` maps to category `"volatility"` and timestamp `"20:12:07"`.
- The system must parse filenames:
  - Extract the category name (capitalized in UI).
  - Extract the timestamp.
  - Fall back gracefully to the file creation/modification time and prefix name if files are not named with the standard format.
- Sort snapshots chronologically within each category.

### 3. Navigation & Tabbed Layout (Option A)
- **Category Navigation**: Display a left sidebar listing all available categories today with a badge count indicating the number of captures.
- **Captures Timeline**: Display a vertical chronological timeline on the right side showing all captures of the active category, connected by a continuous visual thread.
- **Auto-Follow Mode**: Provide a toggle (enabled by default) that automatically switches the view to focus the newest capture when background updates occur. If the trader manually clicks an older snapshot on the timeline, auto-follow is temporarily disabled.

### 4. Interactive Viewport
- Display the selected image centered on a dark canvas.
- Provide a toolbar with:
  - **Zoom controls**: `+` and `-` scaling from 50% to 300%.
  - **Aspect fit modes**: 
    - `Fit`: Stretch/shrink image to fit the container boundaries (aspect ratio preserved).
    - `Fill`: Make image fill the viewport container (cropped if necessary).
    - `Original`: Display image in original high-res dimensions (with scrollbars).

### 5. Desktop Integration & Launchers
- Provide a quick launcher button labeled "Snapshots Board" in the shared Title Bar.
- Add an entry in the application menu (`View -> Show Snapshots Board`).
- Bind keyboard shortcut `CmdOrCtrl+3` to toggle the window.

## Architecture & IPC Specification

### 1. IPC Invocation Channels
- `snapshots:list` -> Returns metadata array of snapshots found in today's folder.
  ```typescript
  interface SnapshotMeta {
    filename: string;
    category: string;
    timestamp: string; // HHMMSS
    mtime: number;
  }
  ```
- `snapshots:read` -> Accepts `filename` and reads local file binary, returning a base64 encoded data URL:
  ```typescript
  (filename: string) => Promise<string> // data:image/png;base64,...
  ```

### 2. IPC Events
- `snapshots:updated` -> Broadcasts the updated file metadata list to the renderer whenever today's folder contents change.

### 3. File Watcher
- Initialize a Node `fs.watch` listener in the main process pointing to today's date folder.
- Trigger folder scans on additions or deletions, broadcasting the results to all open window processes.
